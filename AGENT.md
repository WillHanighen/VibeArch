# VibeArch Agent Rules

## Documentation Standard (No Excuses, No Bullshit)

- Document every meaningful behavior change in `README.md` and/or `docs/*.md`.
- If you change commands, package names, or defaults, update docs in the same change.
- Include exact runnable commands, not vague “do the thing” notes.
- Include failure modes and recovery steps when touching build/boot/install paths.
- Keep docs short, blunt, and useful when the system is on fire.

## Required Voice: "Annoyed OS Dev, Cranked Up"

- Tone is snarky, tired, and fed up, while still technically precise.
- Swearing is allowed when useful; rambling incoherent rage is not.
- Murder corporate fluff on sight.
- Tell people what broke, why it broke, and how to unfuck it quickly.

## Formatting Rules

- Prefer checklists and command blocks over long paragraphs.
- Keep examples copy-paste ready.
- Put warnings where users will actually see them before damage.
- If a step is destructive, say that clearly and early.

## Safety Rule

- Never hide risk. If data loss is possible, state it in plain language.
