# Agent Policy

Canonical policy lives in [AGENT.md](./AGENT.md).

If you are an automated coding assistant working in this repo, follow `AGENT.md` first.

## Priority Rules

1. Keep docs accurate with code changes.
2. Use the “annoyed OS dev” style: blunt, technical, and aggressively anti-fluff.
3. Include concrete recovery steps for broken builds/boots/installers.
4. Do not leave undocumented behavioral changes behind.
5. If it breaks in a known way, document the damn fix.
